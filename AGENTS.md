# Цель проекта.
В данном проекте мы будем делать браузер, который парсит данные из других поисковиков (вайлдбериз и озон), затем объединяет их и выдает пользователю. (т.е. пользователь вводит например "кроссовки найк" - этот запрос уходит в 2 приложения (те, от которых пользователь дал доступы, но позже добавляет возможность добавления других МП (маркет плейсов), например "Яндекс Директ" и др.). Так вот Нужно как-то Одинаковые позиции "схлопывать в одну", по возможности. А остальное выдавать в общей выдаче (одни и те же товары - одна карточка, разные цены с логотипом каждого МП).

# Основные правила:
1. Нужно максимально надёжное решение. Код должен быть мобильным. Похожие и повторяющиеся алгоритмы должны быть объединены в функции/блоки, при этом основные функции/блоки (+ общие по логике) должны быть вынесены в отдельные файлы.
2. Стараться писать параметры вызываемых функций в одну строку.
3. Так же при обновлении функций/модулей желательно скидывать их целиком (если они не большие). Если большие, то четко указывать место в коде/модуле, где обновить нужный кусок (между чем и чем, что бы я понял, куда у себя в коде вставлять. без абстракций).
4. Для всего кода в рамках проекта нужно вести детальное + основное логирование (строго через "print"). Модуль с логированием загружен в файлы проекта. 
Главные правила логирования:
- Всё логирование должно быть строго через терминал (ТОЛЬКО функция print) + дублирование 1:1 этих логов в терминале в лог файл txt (путь к файлу txt задается в main). Этот txt файл каждый раз перезаписывается при запуске скрипта, что бы не тянуть старые логи.
- Всё логирование должно быть на русском языке, с использованием разных эмодзи (для отличий).
- В main файле будет настройка, позволяющая включать/выключать детальное логирование. В обычном режиме в консоль + txt файл идут только основные логи. При включения детального режима - идут все логи. В связи с этим, в любом предложеном коде в рамках проекта всегда максимально детально расписывай логирование для детального режима (что бы у нас было максимум информации, если пойдет что-то не так => я мог просто скинуть тебе лог и ты с первого раза мог понять, в чем проблема), а основное логирование задавай только в ключевых узлах кода.
5. "Не скупись" на детальное логирование. То есть добавляй детальное логирование по максимуму (в разумных пределах, там где это будет хоть чуть-чуть полезно). Не забывай про детальное логироваание при генерации любого кода для проекта.
